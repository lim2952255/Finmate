## 현재 큰 흐름

- 로그인/회원가입은 세션 기반 인증과 인터셉터 중심으로 구성되어 있다.
- 일반 계좌는 계좌 개설, 대표계좌, 계좌이체, 거래내역, 이체한도, 동시성 제어까지 기본 뼈대가 잡혀 있다.
- 증권 계좌는 계좌 개설, 목록, 대표 증권계좌, 홈 요약, 예수금 입금/출금, 예수금 입출금 내역 조회까지 기본 흐름을 구현했다.
- 일반 계좌와 증권 계좌의 계좌번호 중복은 `AccountNumberRegistry`로 공통 관리한다.
- 일반 계좌이체와 증권 예수금 이동은 화면과 서비스 흐름을 분리해서 관리한다.
- 주식 도메인은 KIS 종목 마스터 동기화, 종목 검색/관심종목, 종목 상세, 일봉 on-demand 동기화, SVG 캔들 차트, 거래량/거래대금 TOP10 캐시까지 구현했다.
- 거래내역, 예수금 거래내역, 종목 검색/관심종목 화면은 `PaginationInfo`로 페이징 정보를 공통화했다.
- KIS 연동은 access token 메모리 캐시, 호출 간격 제어, 호출 제한 재시도, Redis 랭킹 캐시를 사용한다.
- KIS 외부 API 장애나 점검 시 전용 503 응답/에러 화면 처리는 아직 보완 대상이다.
- 운영체제에서 배운 개념을 서비스 설계에 적용했다.
  - 일봉 데이터는 페이지 폴트처럼 “필요한 시점에 부족한 구간만 가져오는” on-demand 방식으로 동기화한다.
  - 계좌이체와 예수금 이동은 데드락 방지를 위해 자원 lock 획득 순서를 고정한다.
  - 거래량/거래대금 TOP10은 Redis를 캐시 계층으로 두고 장중에만 주기적으로 갱신한다.
